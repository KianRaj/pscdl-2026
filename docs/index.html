Abstract

The PSCDL 2026 challenge asks for pixel-wise change masks for long-duration object encroachments in fixed-camera surveillance video. The task is harder than naive background subtraction: people pass through and stand still, lighting drifts, and objects must be flagged only after persisting for p seconds and forgotten after c seconds. We combine a custom pixel-level finite-state machine on a dual background (long-term median + slow adaptive) with a trained Siamese ResNet50 as a secondary low-contrast signal, a YOLOv8m person filter, motion-stability and recency filters, and SAM3 for precise mask boundaries. The pipeline runs on a single mid-range GPU, processes a 5–10 minute video in under five minutes, and achieves 0.8465 overall F1 on the released sample set — with 1.000 precision on one of the videos.

The pipeline takes a single video and a pair of parameters (p, c) and emits one binary mask per second.

Pipeline components

Dual-Background PFSM

A static median background (first 15 s) plus a slow adaptive running average. A pixel is "changed" only when it differs from both — gradual lighting drift gets absorbed by the EMA, while a real new object differs from both.

Siamese ResNet50

Trained on VL-CMU-CD + PSCD (4 425 pairs, val F1 = 0.7989). Provides a learned change signal that catches low-contrast objects that fall below frame-diff thresholds.

YOLOv8m person filter

Detects people every frame. Blobs whose area is > 55% covered by a person bbox are rejected — eliminates stationary-person false positives without GT supervision.

Motion-stability filter

True abandoned objects show < 2 px frame-to-frame change. Standing people breathe and shift (3–6 px). The 6 px threshold cleanly separates them.

Recency filter

A blob's newly_pct — the fraction of its pixels with hit_count just above P — must exceed 30%. Long-standing scene artifacts have hit_count ≫ P and fail this test.

SAM3 segmentation

Meta's facebook/sam3 with a size-aware text prompt and the hit-region centroid as the seed point. Margin-expanded bbox lets SAM3 capture the whole object even when only one part is initially detected.

From F1 = 0.06 to F1 = 0.85 — key fixes

The pipeline went through many iterations. The biggest gains came from a few specific bug fixes:

  1. The F1 metric bug — compute_f1 returned 0 when both prediction and GT were blank instead of 1. Most of every video is blank intervals; the bug erased ~80% of our score. Single change: +0.71 F1.
  2. Background-window fix — we were building the median background from the first 100 seconds. The bag in video_1 arrives at t=28 s, so 72% of background frames already contained it. Cutting to first 15 s fixes this.
  3. MOG2 → PFSM — OpenCV's MOG2 absorbs stationary objects into the background within ~30 seconds, which is the opposite of what we want for persistent-change detection.
  4. Replaced ByteTrack with a centroid tracker — ByteTrack's min_hits = 3 requires three consecutive detections. In busy scenes, one person walking past breaks the streak and delays detection by 30+ seconds.
  5. Double-P timing fix — previously the persistence counter and the state-machine both waited P seconds, pushing active windows 2×P seconds late.
  6. Six-stage filter cascade — arrival-time, recency, motion-stability, YOLOv8 person, spatial cooldown, upper-frame centroid — each addresses a different false-positive mode discovered through visual inspection.

Every edit is documented in the repository README changelog.

Usage

The challenge entry point is a single function with the required signature:

from submit import generate_mask

generate_mask(
    p=60,                              # persistence threshold (seconds)
    c=90,                              # cooldown period (seconds)
    video_path="path/to/video.mp4",
    output_dir="output_masks"          # mask_0001.png, mask_0002.png, …
)

Both p and c flow through the entire pipeline — no hard-coded constants. The blind-evaluation team can change them at will.
